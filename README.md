# Система отметки посещаемости

Веб-страница для отметки присутствия студентов на лекции через QR-код. Студент сканирует QR, вводит ФИО и номер группы — данные сохраняются в Google Sheets. Без бумажных листков и переклички.

## Как это работает

Преподаватель показывает QR-код на проекторе в начале пары. Студент сканирует телефоном, вводит ФИО и номер группы, нажимает «Отметиться». Страница проверяет геолокацию и временное окно, после чего записывает отметку в таблицу.

## Защита от злоупотреблений

| Сценарий | Защита |
|---|---|
| Открыл страницу повторно в том же браузере | `localStorage` — форма не открывается |
| Очистил `localStorage` и попробовал снова | `device_id` на сервере — дубль отклоняется |
| Открыл другой браузер на том же телефоне | Fingerprint устройства — дубль отклоняется |
| Передал QR другу домой | Геолокация — далеко от корпуса не пройдёт |
| Подозрительная активность | IP, провайдер, User-Agent, экран видны в таблице для ручного контроля |

## Как вычисляется ID устройства

**Device ID** — случайная строка вида `dev_x7k2m9...`, которая генерируется один раз при первом открытии страницы и сохраняется в `localStorage` браузера:

```javascript
id = 'dev_' + Math.random().toString(36).slice(2) + Date.now().toString(36)
localStorage.setItem('att_device_id', id)
```

При каждой следующей отметке браузер берёт уже сохранённый ID и отправляет его на сервер. Apps Script проверяет: если такой ID уже есть в таблице для той же даты и предмета — отметка отклоняется.

Особенности:
- Привязан к конкретному браузеру, а не к устройству — в другом браузере будет другой ID
- Очищается вместе с данными браузера (очистка кэша, режим инкогнито)
- Именно поэтому используется совместно с Fingerprint

## Как вычисляется Fingerprint

**Fingerprint** — хэш, который вычисляет библиотека [FingerprintJS](https://github.com/fingerprintjs/fingerprintjs). Она собирает около 50 характеристик браузера и устройства и превращает их в короткую строку, например `a1b2c3d4e5f6...`.

В число характеристик входят: модель GPU и его рендеринг, разрешение и глубина цвета экрана, установленные шрифты, User-Agent, язык, часовой пояс, поддерживаемые аудио/видео форматы, параметры Canvas и WebGL и другие технические параметры.

Ключевое свойство: разные браузеры на одном телефоне дают очень похожий или одинаковый fingerprint, потому что характеристики железа (GPU, экран) не меняются при смене браузера. Это закрывает дыру, которую не закрывает Device ID.

Особенности:
- Не 100% надёжен — на разных телефонах одной модели fingerprint может совпасть
- Работает как второй рубеж защиты после Device ID
- Использует только открытую бесплатную версию библиотеки (без облачного API)

## Файлы

- `index.html` — страница для студентов, размещается на GitHub Pages
- `Code.gs` — скрипт для Google Apps Script, принимает данные и пишет в таблицу

## Установка

### 1. Google Apps Script

1. Откройте Google Таблицу (или создайте новую)
2. Расширения → Apps Script
3. Вставьте содержимое `Code.gs`, сохраните
4. Нажмите «Развернуть» → «Новое развёртывание»
5. Тип: **Веб-приложение**, выполнять как: **Я**, доступ: **Все**
6. Скопируйте полученный URL

> При последующих обновлениях скрипта: «Развернуть» → «Управление развёртываниями» → карандаш → «Новая версия» → Сохранить. URL не меняется.

### 2. Настройка страницы

Откройте `index.html` и отредактируйте блок `CONFIG` в начале скрипта:

```javascript
const CONFIG = {
  LAT: 57.8194,           // Широта корпуса
  LNG: 28.3318,           // Долгота корпуса
  RADIUS_M: 150,          // Допустимый радиус в метрах
  START_TIME: '10:00',    // Время начала пары (ЧЧ:ММ)
  WINDOW_MIN: 15,         // Минут для отметки с начала пары
  APPS_SCRIPT_URL: '...', // URL из шага 1
  SUBJECT: 'Технологии программирования',
  GROUPS: ['4421', '4426', '4427', '4428'],
};
```

Координаты корпуса: откройте Google Maps → найдите корпус → правая кнопка мыши → «Что здесь?».

### 3. Публикация на GitHub Pages

1. Создайте репозиторий на GitHub (Public)
2. Загрузите `index.html` (переименованный из `attendance.html`) и `Code.gs`
3. Settings → Pages → Branch: `main`, папка: `/ (root)` → Save
4. Через 1–2 минуты появится адрес вида `https://ваш-логин.github.io/имя-репозитория/`

### 4. QR-код

Сгенерируйте QR-код на полученную ссылку на любом сервисе (например, qr-code-generator.com) и добавьте в презентацию.

## Перед каждой парой

Обновите `START_TIME` в `CONFIG` на актуальное время начала пары и залейте файл в репозиторий. GitHub Pages обновится автоматически в течение минуты.

Если ведёте несколько групп с разным расписанием — удобно держать отдельный файл для каждой группы и показывать соответствующий QR.
