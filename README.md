# Система отметки посещаемости

Веб-страница для отметки присутствия студентов на лекции через QR-код. Студент сканирует QR, вводит ФИО и номер группы — данные сохраняются в Google Sheets. Без бумажных листков и переклички.

## Как это работает

Преподаватель показывает QR-код на проекторе в начале пары. Студент сканирует телефоном, вводит ФИО и номер группы, нажимает «Отметиться».

**Время пары** берётся с сервера Google Apps Script (`GET ?action=time`), а не с часов телефона — так нельзя обойти окно отметки, просто переведя время на устройстве.

**Геолокация** — серия из пяти независимых замеров (с паузой ~1,5 с между ними). По ним считаются средние координаты, расстояние до корпуса, разброс координат и точности, признаки, похожие на Fake GPS, а также согласованность GPS с геопозицией по IP.

После проверок данные отправляются в таблицу через веб-приложение Apps Script.

## Защита от злоупотреблений

| Сценарий | Защита |
|---|---|
| Открыл страницу повторно в том же браузере в тот же день для той же пары | `localStorage` по ключу предмет + список групп + дата — форма не открывается |
| Очистил `localStorage` и попробовал снова | `device_id` на сервере — дубль отклоняется |
| Открыл другой браузер на том же телефоне | Fingerprint — дубль отклоняется |
| Передал QR другу домой | Геолокация — далеко от корпуса не пройдёт; отметка с не-российского IP блокируется (если страна по IP определена) |
| Несколько человек с одного Wi‑Fi | Не блокируется; в таблице помечается повтор IP и текстом в «Флаги» |
| Подозрительная геолокация или не телефон | Флаги в таблице: Fake GPS‑подобные признаки, большое расхождение GPS и IP, отметка не с телефона |

## Как вычисляется ID устройства

**Device ID** — случайная строка вида `dev_x7k2m9...`, которая генерируется один раз при первом открытии страницы и сохраняется в `localStorage` браузера:

```javascript
id = 'dev_' + Math.random().toString(36).slice(2) + Date.now().toString(36)
localStorage.setItem('att_device_id', id)
```

При каждой следующей отметке браузер берёт уже сохранённый ID и отправляет его на сервер. Apps Script проверяет: если такой ID уже есть в таблице для той же **даты**, **предмета** и **строки групп курса** (как в `CONFIG`) — отметка отклоняется.

Особенности:
- Привязан к конкретному браузеру, а не к устройству — в другом браузере будет другой ID
- Очищается вместе с данными браузера (очистка кэша, режим инкогнито)
- Именно поэтому используется совместно с Fingerprint

## Как вычисляется Fingerprint

**Fingerprint** — хэш, который вычисляет библиотека [FingerprintJS](https://github.com/fingerprintjs/fingerprintjs) (v4 с CDN). Она собирает характеристики браузера и устройства и превращает их в строку `visitorId`.

Ключевое свойство: разные браузеры на одном телефоне часто дают схожий fingerprint, потому что характеристики железа не меняются при смене браузера.

Особенности:
- Не 100% надёжен — на разных телефонах одной модели fingerprint может совпасть
- Работает как второй рубеж защиты после Device ID
- Используется открытая версия библиотеки (без облачного API)

## Файлы

- `index.html` — страница для студентов (можно публиковать на GitHub Pages как есть или переименовать в `index.html`, чтобы открывалась по корню сайта)
- `Code.gs` — скрипт Google Apps Script: приём POST, запись в таблицу, проверка дублей, эндпоинт времени для клиента

## Установка

### 1. Google Apps Script

1. Откройте Google Таблицу (или создайте новую)
2. Расширения → Apps Script
3. Вставьте содержимое `Code.gs`, сохраните
4. Нажмите «Развернуть» → «Новое развёртывание»
5. Тип: **Веб-приложение**, выполнять как: **Я**, доступ: **Все**
6. Скопируйте полученный URL

> При последующих обновлениях скрипта: «Развернуть» → «Управление развёртываниями» → карандаш → «Новая версия» → Сохранить. URL не меняется.

### 2. Настройка страницы

Откройте `index.html` и отредактируйте блок `CONFIG` в скрипте:

```javascript
const CONFIG = {
  LAT: 59.838937,         // Широта корпуса
  LNG: 30.104297,         // Долгота корпуса
  RADIUS_M: 200,          // Допустимый радиус в метрах
  START_TIME: '19:00',    // Время начала пары (ЧЧ:ММ) — по серверу Apps Script
  WINDOW_MIN: 120,        // Минут для отметки с начала пары
  APPS_SCRIPT_URL: '...', // URL из шага 1 (заканчивается на /exec)
  SUBJECT: 'Технологии программирования',
  GROUPS: ['4421', '4426', '4427', '4428'],
};
```

Координаты корпуса: Google Maps → точка на здании → «Что здесь?» или координаты из картографического сервиса.

### 3. Публикация на GitHub Pages

1. Создайте репозиторий на GitHub (Public)
2. Загрузите `index.html` и при необходимости `Code.gs` (для справки в репозитории)
3. Чтобы страница открывалась по адресу `https://…github.io/репозиторий/` без `/index.html`, переименуйте файл в **`index.html`**
4. Settings → Pages → Branch: `main`, папка: `/ (root)` → Save
5. Через 1–2 минуты появится URL вида `https://ваш-логин.github.io/имя-репозитория/`

### 4. QR-код

Сгенерируйте QR на итоговую ссылку (любой генератор QR) и вставьте в слайд.

## Таблица посещаемости

Данные пишутся на лист **«Посещаемость»** (создаётся автоматически с заголовками). Колонки в порядке записи:

| # | Колонка | Описание |
|---|---------|----------|
| 1 | Дата | По серверному времени Apps Script |
| 2 | Время | По серверному времени |
| 3 | ФИО | Ввод студента |
| 4 | Группа студента | Ввод студента |
| 5 | Предмет | `CONFIG.SUBJECT` |
| 6 | Группы курса | `CONFIG.GROUPS` через запятую |
| 7–9 | Широта, долгота, расстояние (м) | Усреднённая геолокация и до корпуса |
| 10 | Метка времени (ISO) | Серверное время на момент отметки |
| 11–16 | Точность GPS, дисперсии координат и точности, σ интервалов, ср. скорость (км/ч), σ скорости | Анализ серии замеров |
| 17–18 | Высота есть, Heading есть | Наличие полей в замерах |
| 19 | Подозрительно | «ДА» / «нет» (эвристики + не телефон + GPS vs IP) |
| 20 | Флаги | Текстовые пояснения; при повторе IP на паре — пометка от сервера |
| 21 | GPS vs IP (км) | Расстояние между позицией по GPS и по гео IP (если известно) |
| 22 | Тип устройства | телефон / планшет / компьютер (эвристика) |
| 23 | ID устройства | `localStorage` |
| 24 | Fingerprint | FingerprintJS |
| 25–29 | IP, страна, город, регион, провайдер | В основном [ipapi.co](https://ipapi.co); при сбое IP может подтянуться через ipify |
| 30–41 | User-Agent, платформа, языки, экран, часовой пояс, CPU, RAM, тач, соединение, скорость, батарея | Браузерные API |

> Поля вроде RAM, типа соединения, батареи поддерживаются не всеми браузерами — тогда может быть `unknown`. Страна по IP: если не удалось определить (`unknown`), проверка «только Россия» не блокирует отправку.

## Серверная логика (кратко)

- **Дубликаты** по `device_id` и по `fingerprint` (если не `unknown`) — ответ `duplicate`, в ответе передаётся ФИО уже отметившегося.
- **Повтор IP** на той же дате/предмете/группах курса у **другого** человека — запись не блокируется; в данные добавляются флаги и `Подозрительно: ДА`.

## Перед каждой парой

Обновите `START_TIME` и при необходимости `WINDOW_MIN` в `CONFIG`, зафиксируйте изменения в репозитории. GitHub Pages подтянет обновление обычно в течение минуты.

Если расписание разное — удобно держать отдельные ветки/файлы или отдельные репозитории с разным `CONFIG` и разными QR.
